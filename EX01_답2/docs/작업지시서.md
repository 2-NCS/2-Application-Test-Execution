## ① 작업지시서(문제)

미완성 EX01(S-Library) 프로젝트는 도서 대출 기능의 본문 코드는 제공되지만 테스트 코드가 비어 있고
일부 로직에 결함이 포함되어 있다. 단위 테스트(JUnit5·Mockito), 슬라이스 테스트(@DataJpaTest·
@WebMvcTest), 통합 테스트(@SpringBootTest)를 작성하여 대출/반납/연체/재고 기능이 요구사항을
충족하는지 검증하고, 테스트로 드러난 결함을 결함 관리대장에 기록한 뒤 원인을 분석하여 수정하고
회귀 테스트로 부작용이 없음을 확인하시오. 테스트 실행 결과(수정 전·후)와 산출물 문서를 제출하시오.
(도메인 코드·DB seed·결함이 포함된 서비스 제공)

---

## ② 내용(설명)

### [확인]
- 도메인     : 도서 대출 (Book / Lend) — 대출·반납·연체 판정·대출가능 부수 조회
- 테스트 대상 : LibraryServiceImpl(비즈니스 로직), BookRepository(JPA), BookController(REST)
- 테스트 유형 : 단위(Mockito) / 슬라이스(@DataJpaTest·@WebMvcTest) / 통합(@SpringBootTest)
- 실행 환경  : 인메모리 H2 (별도 DB 설치 불필요), 포트 8090
- 결함 관리  : 실패 테스트 → 결함 관리대장 기록(유형·심각도·우선순위·재현) → 원인분석 → 수정 → 회귀
- 형상 관리  : 결함 조치마다 Git 커밋으로 이력 관리
- 환경       : JDK 21 / Spring Boot 3.5.x / JUnit5 / H2(test)
- 제공물     : 도메인(Entity/Repository/Service/Controller/Dto/Exception), application.properties,
               data.sql(seed 3건), 비어 있는 테스트 클래스 4종(TODO)
- 학생 구현  :
   · LendServiceTest        : 정상/재고0 경계/미존재 도서/중복 반납/연체 경계
   · BookRepositoryTest     : ISBN 조회·존재 확인 (@DataJpaTest)
   · BookControllerTest     : 대출 200 / 입력 검증 400 (@WebMvcTest + @MockitoBean)
   · LibraryIntegrationTest : 대출→재고 반영 흐름 (@SpringBootTest)
   · 산출물 4종 + 결함 수정(src/main)

### [핵심 수행 요구사항]

1) 테스트 작성(능력단위요소 ①)
   - 단위 : 정상 대출(재고 1 감소·dueAt 설정), 재고 0 경계, 미존재 도서 예외, 중복 반납 차단, 연체 경계값
   - 슬라이스 : @DataJpaTest 로 ISBN 조회/존재 확인, @WebMvcTest 로 응답 상태·검증 400
   - 통합 : @SpringBootTest 로 대출 후 대출가능 부수가 0 으로 반영되는지 확인
   - 테스트케이스_명세서·테스트_실행결과서 작성

2) 결함 조치(능력단위요소 ②)
   - 실패한 테스트가 가리키는 결함을 식별하여 결함 관리대장에 기록(유형·심각도·우선순위·재현 절차)
   - 원인을 분석하고 src/main 코드를 수정
   - 전체 테스트를 다시 실행하여 모두 통과함을 확인(회귀 테스트로 부작용 없음 확인)
   - 결함조치_결과서 작성, 결함별 Git 커밋
